Getting Started

Prerequisites

On Linux or OSX, you're best to install with your package manager.

On OSX, I recommend {{=A('Homebrew', _href='http://brew.sh')}}.

On Windows, follow the links below.

Install the following:
  • {{=A('Python 2.7', _href='https://www.python.org/downloads/')}}
  • {{=A('git', _href='http://git-scm.com/download/win')}}
  • {{=A('cmake', _href='http://www.cmake.org/download/')}} or greater

Creating a Project

Gaen can build and run stand alone. In most cases, however, it is preferrable to create a project based on Gaen, whereby you can have some project specific C++ as well as your project specific Compose scripts living side by side with the Gaen source code.

Create a new project as follows. In this example, we'll create a project called 'foo'.

  1. Make a new blank directory for your project, and cd into it. {{=PRE('''C:\\code> mkdir foo C:\\code> cd foo ''')}}
  2. Clone the Gaen repository and run start_project.py. {{=PRE(r'C:\code\foo> git clone https://github.com/lachlanorr/gaen.git')}}
  3. You will be prompted to continue, press 'Y' to create the project. {{=PRE('''Starting new project 'foo' in current directory... Continue? [Y/n] Y Gaen project 'foo' created in current directory. Run bootstrap.bat to generate Visual Studio solution. ''')}} The project name is inferred from the current working directory. When completed, several files will be placed within the foo directory.
  4. Run bootstrap.bat to compile initial compose scripts as well as run cmake and create the Visual Studio solution. {{=PRE(r'C:\code\foo> bootstrap.bat')}}
  5. Open the solution. {{=PRE(r'C:\code\foo> build\win64\foo.sln')}}
On Linux or OSX, these instructions are nearly identical, except you need to run bootstrap.sh. On OSX, you'll open the generated Xcode project. On Linux, you'll run make against the generated Makefile.
