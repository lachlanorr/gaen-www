{{=T('Creating a Project')}}

By itself, Gaen can build an executable and run stand alone. In most cases, however, it is preferrable to create a project based on Gaen, whereby you can have some project specific C++ as well as your project specific Compose scripts living side by side with the Gaen source code.

Create a new project as follows. In this example, we'll create a project called 'foo'.

  1. Make a new blank directory for your project, and cd into it. {{=PRE(r'''C:\code> mkdir foo C:\code> cd foo ''')}}
  2. Clone the Gaen repository and run start_project.py. {{=PRE(r'C:\code\foo> git clone https://github.com/lachlanorr/gaen.git')}}
  3. You will be prompted to continue, press 'Y' to create the project. {{=PRE('''Starting new project 'foo' in current directory... Continue? [Y/n] Y Gaen project 'foo' created in current directory. Run bootstrap.bat to generate Visual Studio solution. ''')}} The project name is inferred from the current working directory. When completed, several files will be placed within the foo directory.
  4. Run bootstrap.bat to compile initial compose scripts as well as run cmake and create the Visual Studio solution. {{=PRE(r'C:\code\foo> bootstrap.bat')}}
  5. Open the solution. {{=PRE(r'C:\code\foo> build\win64\foo.sln')}}
